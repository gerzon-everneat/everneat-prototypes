Bundle implementation guide

Exactly how to build the Surface Cleaner bundle (Product Page V2 + subscribe) on the Shopify Storefront API + your synced CMS. File-by-file, with the real code changes.

Overview

The bundle is a soft bundle: the product page adds a bottle and a concentrate refill as two real Shopify cart lines in one cartLinesAdd. Nothing is a "Shopify bundle product" or Function.

LayerOwns
Shopify (Storefront API)Products, variants, options, prices, inventory, selling plans, cart, hosted checkout
Your CMS · product-attribute-apiPer-product editorial (scent blurbs, benefits, where-to-use…) + the bundle pairing, keyed by Shopify product ID. No prices/stock.
Storefront (React)Reads pairing → re-hydrates live from Shopify → renders V2 page → adds 2 lines

Five phases. Each below has Goal, the files to touch, the exact steps/code, and a Verify check.

0 · Prerequisites (before any code)

Shopify — subscription on the refill

The "Subscribe −15% / every N weeks" only exists if the refill product has a selling plan. In Shopify admin (native Shopify Subscriptions or your subscription app):

  • Add the refill to a selling-plan group with a 15% percentage discount.
  • Delivery-interval options: 4 / 6 / 8 / 12 weeks.
  • Tag the refill subscription (the app's getProductSellingPlans.ts queries query:"tag:subscription").
If subscriptions aren't set up yet — ship the bundle one-time only first (skip the Subscribe row in Phase 3), and turn it on later. Nothing else changes.

CMS — add the pairing to the product-attribute payload

The PDP already fetches per-product CMS content via getFiberyProductByProductId(product.id) (the product-attribute-api edge function, keyed by the Shopify product ID). Author the pairing as a field on that same payload — right where refillsAndKits already lives:

// CMS "bundle" record (editorial-verse / Supabase)
{
  host_handle:    "surface-cleaner",        // the PDP this bundle shows on
  plastic_handle: "surface-cleaner",
  glass_handle:   "surface-cleaner-glass",
  refill_handle:  "surface-cleaner-concentrate-refill-in-aluminum-bottle-copy",
  default_refill_size: "8",
  copy: { title: "Bottle + Refill", blurb: "…" },
  active: true
}

Verify the refill returns a non-empty sellingPlanGroups from Shopify, and a /product/attribute/{id} response includes a bundle field for the host product.

Add structured fields to productFields

In query.ts, inside the variants(first: N){ edges { node { … } } } block add selectedOptions, and at the product level add options. (sellingPlanGroups is already in productFields — subscriptions are ready; no change needed there.)

3

Resolve variants by option, not title

In useVariantSelector.ts, match the chosen scent/size against selectedOptions instead of parsing node.title against the hardcoded allScents + oz-regex. Return "unavailable" when there's no exact match — never fall back to a default variant.

const findVariant = (scent, size) =>
  variants.find(v => {
    const o = Object.fromEntries(v.node.selectedOptions.map(s => [s.name, s.value]))
    return o.Scent === scent && (size ? o.Size === size : true)
  }) ?? null   // null = disable that combination
Heads up — title-parsing is used elsewhere too (cart line labels, card components). Scope this change to the bundle flow first, or migrate call-sites deliberately.

Verify log variants[].selectedOptions for the bottle & refill; confirm scent/size resolve and the refill returns sellingPlanGroups.

2 · CMS pairing + resolver

Goal Fetch the bundle pairing from the CMS and hydrate every member live from Shopify.

src/features/common/api.ts (existing getFiberyProductByProductId) · src/features/shop/useFetchDataHook.tsx (already exposes it as fiberyProductQuery) · new: resolveBundle()

1

Read the pairing — no new endpoint

The PDP already calls getFiberyProductByProductId(product.id) and exposes it as fiberyProductQuery. The pairing rides on that payload — just read value.bundle:

// already fetched on the product page (useFetchDataHook)
const bundleDef = fiberyProductQuery.data?.value?.bundle
// → { refillHandle, plasticHandle, glassHandle, defaultRefillSize, active } | undefined
2

Resolver

Reuse the RefillsAndKits.tsx Promise.all(getProductByHandle) pattern — but validate each member (RefillsAndKits does not).

async function resolveBundle(bundleDef, userTags) {
  if (!bundleDef?.active) return null                  // no pairing → normal PDP
  const [plastic, glass, refill] = (await Promise.all([
    getProductByHandle(bundleDef.plasticHandle),
    getProductByHandle(bundleDef.glassHandle),
    getProductByHandle(bundleDef.refillHandle),    // Shopify = live variants, price, sellingPlanGroups
  ])).map(r => r.body?.product)

  const ok = (p) => p && p.variants.edges.some(v => v.node.availableForSale)
                  && !shouldExcludeProduct(p, userTags)        // per-viewer veto
  if (![plastic, glass, refill].every(ok)) return null  // hide if any member invalid
  return { plastic, glass, refill }
}

Verify for a product whose CMS payload has value.bundle, the resolver returns hydrated bottle(s) + refill; for an unpublished/OOS member it returns null.

2

The thunk

// src/features/cart/slice.ts
const addBundleToCartAsync = createAsyncThunk("cart/addBundle", async (sel) => {
  const { cartId } = await ensureCart()
  const bundleId = crypto.randomUUID()
  const lines = [
    { variantId: sel.bottleVariantId, quantity: caseQty(sel.bottleProduct),
      attributes: [{ key: "_addedFromBundleId", value: bundleId }] },
    { variantId: sel.refillVariantId, quantity: caseQty(sel.refillProduct),
      sellingPlanId: sel.sellingPlanId || null,
      attributes: [{ key: "_addedFromBundleId", value: bundleId }] },
  ]
  return addCartLines({ cartId, lines })   // one cartLinesAdd → 2 lines
})
3

Per-member quantity from the case_ tag

Don't reuse the hardcoded quantity: isB2B ? 12 : 1 from GeneralButtonAddToCart.tsx — derive each line's quantity from its own product's case_ tag via extractCaseNumber (Surface Cleaner products are all case_12, but the refill could differ).

Verify add the set → cart has 2 lines; the refill line shows the subscription; the hosted checkout total reflects the 15%.

5 · Grouped cart card (deferred — optional polish)

Only if you want the two lines shown as one "set" card. It adds real complexity, so it's last.

  • loadCart.ts — add attributes { key value } to the shared cartFields fragment; cart/types.ts — add attributes to the line type.
  • ItemShop.tsx — group lines by _addedFromBundleId before mapping; switch keys from index → node.id.
  • New BundleCartItem — qty/remove operate on both lines (the mutations already take arrays); handle orphan states.

Edge cases & gotchas

  • Missing / unpublished / OOS member → resolver returns null, bundle hidden (Phase 2).
  • Title parsing is brittle and used elsewhere — migrate to selectedOptions deliberately.
  • CMS sync key — reference a stable Shopify handle/ID so re-syncs don't break the pairing.
  • B2B pricing — the existing wholesale logic (price/2, EXCLUSIVE_DISCOUNT_CODE) still applies to each line; make sure the subscribe discount and wholesale code don't double-count.
  • Hosted checkout — never compute final price client-side beyond display; Shopify is authoritative.
  • No codegen — queries are untyped inline strings; change the GraphQL, the type, and the builder together.

Testing

  1. Per-phase Verify checks above.
  2. E2E: open a bundle PDP → pick scent/material/refill size → choose Subscribe + frequency → Add the set → confirm 2 cart lines, refill on subscription, correct totals → continue to checkout and confirm the 15% holds.
  3. Negative: unpublish the refill → bundle disappears; pick a scent the refill lacks → that option disables (no silent fallback).

Rollout

  1. Ship one-time-only if subscriptions aren't configured yet; enable Subscribe when the selling plan exists.
  2. Gate each bundle with the CMS active flag (instant on/off, no deploy).
  3. Start with one product (Surface Cleaner) → validate → roll the pattern to others by adding CMS records.
